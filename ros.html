ROS ROBOT

Before graduating from high school, we were tasked with creating a final "capstone" project for ourselves to represent our interests before our next step in life. I decided to design, manufacture, assemble, and program a robot to experiment with a program package called ROS. ROS (Robot operating system) offers a framework to build off that gives unique abilities when compared to a standard robot code-base without ROS. For example, a non-ROS based system would have a large array of files used to make instances of objects that communicate with each (with python its slightly different). With ROS, you're able to create individual nodes that each might accomplish some task.

Using this, I was able to make number of nodes in python, which would otherwise be impossible without ROS, to handle more complicated components. One such example was how I made the drivebase for the robot. Instead of making one python file to handle taking in commmands and translating them to PWM pulses, I made a drivebase node that contained two motor nodes. Each node carried its own way of taking in the information it needed and "translating it" for the next node. This made it much simpler for me to program the robot without an extensive knowledge of advanced programming skills (which I don't have as I specialize in mechanical engineering).

By the end of the project, I had made a robot capable of driving with human control, managing its own position relative to the environment, detecting objects with a cheap version of LIDAR, and navigating around obstacles using that object detection.
